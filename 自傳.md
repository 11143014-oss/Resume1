自傳 — 慈濟大學資訊管理學系 四年級生

我叫陳浩平，現為慈濟大學資訊管理學系四年級學生。自高中時期接觸程式設計以來，我便被資訊技術解決實際問題的能力深深吸引。進入大學後，我選擇以「資料分析與應用」與「系統開發實務」兩大方向作為學習重心，期望將技術能力與使用者需求結合，創造有意義的服務。

在學期間，我修習了資料庫系統、程式設計、資料探勘、人工智慧、系統分析與設計、網頁應用開發與雲端運算等課程，並於大三開始投入專題與實作專案。大四專題為「AI 醫療影像診斷系統」，我主要負責資料前處理、模型訓練與系統整合，使用 TensorFlow 與 YOLOv8 進行影像目標偵測，最終模型在測試資料上的準確率達 96% 以上。透過此專題，我學會了如何面對不完整或雜訊較多的資料、有效地進行交叉驗證與參數調整，並在團隊中扮演溝通協調與技術實作的角色。

此外，我曾在校外實習於 〇〇 科技公司擔任資料分析實習生，負責資料清理、ETL 流程設計與建立 Power BI 儀表板，協助行銷團隊追蹤業務 KPI。實習期間我優化了部分查詢邏輯與資料儲存流程，使查詢效能提升約 30%，也因此更了解企業資料流與實際商業需求的差異。另一段後端開發實習則讓我熟悉 Django RESTful API 的設計與測試流程，增進系統開發的完整觀點。

在課外活動方面，我擔任資訊管理系學會幹部，負責技術分享會的籌辦，曾邀請校內外講者介紹雲端技術與資料工程實務，並協助同學進行專題對接。此外，我也積極參與開源社群，在 GitHub 上有數個小型專案貢獻，內容涵蓋資料分析工具與簡易網頁應用，藉此持續磨練程式設計與文件撰寫能力。

我的優勢在於結合技術與溝通：我能快速吸收新技術並用於專案實作，同時擅長將技術細節轉述給非技術同事或使用者，以利需求確認與產品落地。面對問題時我具備系統性的分析方法，會先從資料品質和需求面切入，再逐步設計解決方案與驗證指標。

未來我期望能加入以資料驅動決策或 AI 應用為主的團隊，擔任資料工程師或資料科學相關職務，從事模型開發、資料平台建置或系統整合，並在實務中持續學習 MLOps 與雲端部署等技術，將研究成果轉化為可運行的產品，為組織創造價值。

最後，我重視職業倫理與團隊合作精神，尊重使用者隱私並樂於在多元文化的環境中互相學習。希望未來有機會能在貴單位或團隊中貢獻所學，並與各位共同成長。
